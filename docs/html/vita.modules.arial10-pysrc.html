109  # By default, Excel displays column widths in units equal to the width 
110  # of '0' (the zero character) in the standard font.  For me, this is 
111  # Arial 10, but it can be changed by the user.  The BIFF file format 
112  # stores widths in units 1/256th that size. 
113  # 
114  # Within Excel, the smallest incrementable amount for column width 
115  # is the pixel.  However many pixels it takes to draw '0' is how many 
116  # increments there are between a width of 1 and a width of 2.  A 
117  # request for a finer increment will be rounded to the nearest pixel. 
118  # For Arial 10, this is 9 pixels, but different fonts will of course 
119  # require different numbers of pixels, and thus have different column 
120  # width granularity. 
121  # 
122  # So far so good, but there is a wrinkle.  Excel pads the first unit 
123  # of column width by 7 pixels.  At least this is the padding when the 
124  # standard font is Arial 10 or Courier New 10, the two fonts I've tried. 
125  # It don't know if it's different for different fonts.  For Arial 10, 
126  # with a padding of 7 pixels and a 9-pixel-wide '0', this results in 16 
127  # increments to get from width 0 (hidden) to width 1.  Ten columns of 
128  # width 1 are 160 pixels wide while five columns of width 2 are 125 
129  # pixels wide.  A single column of width 10 is only 97 pixels wide. 
130  # 
131  # The punch line is that pixels are the true measure of width, and 
132  # what Excel reports as the column width is wonky between 0 and 1. 
133  # The only way I know to find out the padding for a desired font is 
134  # to set that font as the standard font in Excel and count pixels.
